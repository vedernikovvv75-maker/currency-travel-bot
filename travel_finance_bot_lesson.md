# Изучаем прототип финансового помощника для путешествий ↘️

✅  
Финансовый помощник — это бот, который помогает планировать бюджет в поездках, автоматически конвертируя суммы между валютами и отслеживая расходы.

## Пример работы с ботом

1. Выбираем страну выезда — например, Россия (валюта определяется автоматически).
2. Указываем страну назначения — например, Италия.
3. Бот находит актуальный курс (например, 104 рубля за 1 евро).
4. Вводим начальную сумму для поездки (например, 200 000 рублей).
5. Бот автоматически создает путешествие и конвертирует сумму (≈ 2000 евро).
6. Отправляем боту суммы расходов, а он учитывает их и показывает остатки.

### Особенности функционала

- история расходов с детализацией по времени и суммам;
- отображение остатков в двух валютах одновременно;
- возможность временного изменения курса для тестирования;
- автоматическое определение валют по странам.

## 3. Подключение к API и документация

### Изучаем документацию и API ↘️

Переходим к первому этапу — работе с API для получения актуальных курсов валют.

✅  
API (Application Programming Interface) — это интерфейс, который позволяет программам взаимодействовать друг с другом. В нашем случае — получать данные о курсах валют от внешнего сервиса.

Мы будем использовать сервис `exchangerate.host`.

Этот сервис предоставляет:

- данные по курсам валют в реальном времени;
- исторические данные;
- информацию о криптовалютах;
- бесплатный тарифный план.

### Регистрация и получение API-ключа

1. Переходим на сайт exchangerate.host.
2. Нажимаем на кнопку **GET FREE KEY** (получить бесплатный ключ).
3. Открывается панель, в которой проходим регистрацию.
4. После входа попадаем в Dashboard.
5. Копируем наш API-ключ для использования в проекте.

⚠️  
Бесплатный тариф дает 100 запросов в месяц. Этого достаточно для обучения и тестирования.

### Работа с документацией

1. В верхней части сайта (прямо в шапке) находим кнопку **Documentation**.
2. В левом меню ищем раздел **API Features**.
3. Выбираем пункт **Real-time Rates** (курсы в реальном времени).
4. Изучаем пример API-запроса.

✅  
Endpoint — это конкретный адрес (URL), по которому мы отправляем запрос к API для получения определенных данных.

Пример API-запроса из документации:

```text
https://api.exchangerate.host/live
? access_key = ЗДЕСЬ_БУДЕТ_НАШ_КЛЮЧ
& currencies = AUD, EUR, GBP, PLN
```

Сервис автоматически подставляет API-ключ в примеры запросов. Это обычный GET-запрос, который можно выполнить прямо в браузере.

### Пишем модуль для работы с API валют

Изучаем endpoint для реальных курсов:

- Endpoint: `https://api.exchangerate.host/latest`
- Параметры: `base`, `symbols`, `access_key`

Создаем функцию получения курсов в `current_api.py`:

```python
import requests


def get_current_rate(default: str = "USD", currencies: list[str] = ["EUR", "GBP", "JPY"]):
    url = "https://api.exchangerate.host/live"
    params = {
        "access_key": "API-ключ",
        "source": default,
        "currencies": ",".join(currencies),
        # ",".join(currencies) объединяет список в строку через запятую
    }

    response = requests.get(url, params=params)
    data = response.json()
    return data
```

Тестируем функцию:

```python
if __name__ == "__main__":
    print(get_current_rate())
```

Пример с другой базовой валютой:

```python
if __name__ == "__main__":
    data = get_current_rate(default="RUB", currencies=["USD", "EUR", "GBP", "JPY", "CNY"])
    print(data["quotes"])
```

### Добавляем функцию конвертации валют ↘️

Endpoint конвертации: `https://api.exchangerate.host/convert`

Параметры:

- `from` — исходная валюта;
- `to` — целевая валюта;
- `amount` — сумма для конвертации.

Функция конвертации:

```python
def convert_currency(amount, from_currency, to_currency):
    url = "https://api.exchangerate.host/convert"
    params = {
        "access_key": "your_api_key",
        "from": from_currency,
        "to": to_currency,
        "amount": amount
    }

    response = requests.get(url, params=params)
    data = response.json()
    return data
```

Тестируем:

- конвертируем 100 USD в EUR;
- пробуем экзотические валюты;
- убеждаемся, что функция работает корректно.

### Подготавливаем промпт для генерации бота ↘️

Теперь, когда модуль для работы с API готов, можно составить подробный промпт для Cursor, который поможет сгенерировать Telegram-бота.

Пример промпта:

> Напиши Telegram-бота под Python, используя pytelegrambotapi. Этот бот — не просто конвертер валют, а миникошелёк для путешественника, работающий через реальный API api.exchangerate.host. Использовать строго этот API и только его. Все запросы должны идти на endpoint http://api.exchangerate.host/convert с параметром access_key, который хранится в конфигурации (вторая переменная — это Telegram-токен). Бесплатные или тестовые API не использовать.
>
> Для получения данных используй @current_api.py.  
> При старте бот предлагает пользователю создать новое путешествие. Пользователь вводит страну отправления и страну назначения. После этого бот делает запрос к api.exchangerate.host и проверяет, какие валюты доступны для этих стран. Затем показывает текущий курс и спрашивает, подходит ли он. Если пользователь отвечает «нет», бот предлагает вручную ввести курс обмена (например, по местному обменнику). После подтверждения создаётся сущность «путешествие». У пользователя может быть несколько путешествий, между которыми можно переключаться. Каждое путешествие хранит валютную пару, курс и текущий баланс.
>
> Важно: при создании путешествия пользователь вводит начальную сумму в своей домашней валюте (той, из которой он выезжает). Бот сам конвертирует её через api.exchangerate.host и создаёт стартовый баланс в обеих валютах. Это позволяет пользователю сразу увидеть, сколько он получит в валюте назначения.
>
> Дальше логика такая: любое сообщение с числом бот воспринимает как сумму расходов в валюте страны пребывания и пересчитывает её в домашнюю валюту. Например, если активен кошелёк «Китай» и пользователь пишет 100, бот считает, что это 100 юаней, конвертирует в рубли и выводит сообщение: «100 CNY = 1280 RUB». После этого бот показывает inline-кнопки «Учесть как расход? ✅ Да / ❌ Нет». Если нажать «Да», сумма записывается как трата и вычитается из баланса путешествия. Баланс всегда показывается в двух валютах, например: «Остаток: 2400 CNY = 30 720 RUB».
>
> Добавить полноценное inline-меню для навигации без необходимости использовать слэш-команды. Главное меню должно содержать кнопки:
>
> - «Создать новое путешествие»
> - «Мои путешествия»
> - «Баланс»
> - «История расходов»
> - «Изменить курс»
>
> При нажатии на каждую кнопку открывается соответствующий раздел. Слэш-команды `/newtrip`, `/switch`, `/balance`, `/history`, `/setrate` должны остаться как альтернативный способ управления. Все данные хранить локально (в SQLite). Ошибки от API и пользователя (не число, нет валюты, ошибка запроса) должны обрабатываться корректно и сопровождаться дружелюбными сообщениями. Каждый пользователь должен иметь свой собственный набор путешествий.

После отправки промпта Cursor начнет генерировать код бота, создаст модуль `database.py` и основной файл бота. Пока идет генерация, можно параллельно установить зависимости.
